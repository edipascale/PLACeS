11 PLACeS is used to compare various content delivery strategies for
12 Video on Demand and time-shifted IPTV over a LRPON-based network.
13 It was written for academic purposes and it's most likely
14 unfit for usage in any production environment.
15 
16 The project requires Boost libraries version 1.58.0.
17 The latest updates also require the ILOCPLEX libraries from IBM's CPLEX.
18 PLACeS was tested only on Ubuntu and Debian, both 32 and 64 bit.
19 
20 The source is currently being released mainly as a proof of concept;
21 minimal documentation is provided in the form of comments throughout
22 the code.
23 
24 For instructions on which simulation parameters are available and how
25 to configure them, run the program with the --help command line argument.

27 ==========
28 What's new
29 ==========
30 
31 We moved from a download model to a streaming model, where video items
32 are divided into fixed-size chunks and only enough chunks to fill a buffer
33 are fetched at each user. The downside of this approach is that, naturally,
34 simulations are much more cumbersome computationally. Chunking can be
35 disabled by setting the chunk-size parameter ('n') to 0. Note however
36 that this will disable the modelling of zapping behaviour, i.e., a change
37 of channel before the end of the current content item. Users who are
38 interested in modelling such behaviours with no chunk division should
39 keep using the release version 1.0.
